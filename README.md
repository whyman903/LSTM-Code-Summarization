# Assignment 2: Code Summarization via LSTM

## Data Sources & Pre-processing

- **Source:** Public GitHub Java repositories discovered via the GitHub Search API and shallow-cloned locally by `get_data.py`.
- **Reference:** CodeXGLUE is used as a reference for the code-to-text task setup.
- **Training:** ~50,000 Java method–summary pairs.
- **Validation:** 1,000 pairs (full set used for early stopping on BLEU-1).
- **Test:** Set of 1,000 Java code-summary pairs (`provided-files/dataset/test_dataset_tokenized.csv`); the notebook warns if the local copy has fewer rows and continues so you can still run.

**Pre-processing:**
1. Search for public Java repositories on GitHub, then shallow-clone them into a local cache.
2. Walk Java source files while excluding common test / sample / generated directories.
3. Extract methods with an immediately preceding Javadoc block; exclude constructors.
4. Use the Javadoc summary section as the summary, stopping at the first blank line or Javadoc tag (for example `@param`), flatten each method into a single whitespace-normalized line, and lowercase each summary.
5. Filter trivially short methods, overly long methods, trivial getters/setters, and noisy summaries (for example URL-heavy or non-English Javadocs).
6. Build repo-disjoint train and validation splits, then tokenize with CodeT5+ via `get_codet5_embeddings.py`.

## How to Reproduce

1. Open `assignment-2-LSTM.ipynb` and run all cells. Use HPC of Google Colab. 

   - Install dependencies
   - Run `get_data.py` to mine GitHub repositories and prepare the dataset
   - Run `get_codet5_embeddings.py` to tokenize and extract embeddings
   - Train the LSTM with early stopping
   - Evaluate on the held-out test set
