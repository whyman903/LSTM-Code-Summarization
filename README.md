# Assignment 2: Code Summarization via LSTM

## Data Sources & Pre-processing

- **Source:** [CodeXGLUE code-to-text (Java)](https://huggingface.co/datasets/google/code_x_glue_ct_code_to_text) — the CodeSearchNet Java subset.
- **Training:** ~50,000 Java method–summary pairs.
- **Validation:** 1,000 pairs (used for early stopping on BLEU-1).
- **Test:** Instructor-provided held-out set (`provided-files/dataset/test_dataset_tokenized.csv`).

**Pre-processing:**
1. Each Java method is flattened into a single whitespace-normalized line.
2. Each summary is lowercased and whitespace-normalized.
3. Trivially short methods (<5 tokens) or summaries (<3 tokens) are filtered out.
4. Tokenized using CodeT5+ (Salesforce/codet5p-220m) via the provided `get_codet5_embeddings.py`.

## How to Reproduce

1. Open `assignment-2-LSTM.ipynb` in Google Colab (GPU runtime recommended).
2. Run all cells top-to-bottom. The notebook will:
   - Install dependencies
   - Run `get_data.py` to download and prepare the dataset
   - Run `get_codet5_embeddings.py` to tokenize and extract embeddings
   - Train the LSTM with early stopping
   - Evaluate on the held-out test set

## Output Files

| File | Description |
|------|-------------|
| `train_code.txt`, `train_summary.txt` | Training data (generated by `get_data.py`) |
| `val_code.txt`, `val_summary.txt` | Validation data (generated by `get_data.py`) |
| `*.pt` | Tokenized data + embedding matrices (generated by `get_codet5_embeddings.py`) |
| `best_model.pt` | Best model checkpoint (by validation BLEU-1) |
